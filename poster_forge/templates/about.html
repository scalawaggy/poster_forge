About Poster Forge

Version 3.1.0-build.42 (Sub-Mariner Edition)

Poster Forge is an automated multi-function media utility that bridges the gap between your *arr instances and Plex. It manages poster acquisition, dynamically generates and paints custom countdown banners for upcoming media, and performs regular maintenance jobs to keep your libraries pristine.

Core Integrations & Requirements

To fully utilize the Automated Tracking Engine, you must configure your external dependencies properly.

1. Plex Configuration

  • You MUST create a Movie Library called Coming Soon (ordered by release date).
  • Then, add items in that library to a Smart Collection with criteria such as: Movie released since 1900.
  • The app will automatically create the Returning soon collection for TV shows. However, you must manually change its collection order to custom so the app can reorder the items dynamically.
  • Ensure your Poster Forge Docker container has mounted your exact Plex Media directories as defined in Settings.

2. Radarr & Sonarr (The *arrs)

  • Poster Forge connects directly to their APIs to identify upcoming releases.
  • In the Poster Forge Settings, input the API Keys for both Radarr and Sonarr.
  • The system will track movies that have a status of announced or inCinemas, and shows with upcoming episode air dates.

3. Tautulli (Real-Time Triggers)

  • Tautulli is used to catch when a movie or show is finally downloaded and added to Plex.
  • When triggered, Poster Forge's Maintenance Job acts as an assassin, purging the temporary trailer and removing the "Coming Soon" banners.
  • Webhook Setup:
    1. In Tautulli, go to Settings > Notification Agents > Add a Webhook.
    2. Webhook URL: http://[POSTER_FORGE_IP]:8000/api/webhook/tautulli?token=YOUR_SECURE_TOKEN
    3. Triggers: Select Recently Added.
    4. Data (JSON): 
      {
    "action": "added",
    "ratingKey": "{rating_key}",
    "mediaType": "{media_type}",
    "title": "{title}"
}

Security Note

Ensure you configure a Webhook Security Token in the Settings panel. Leaving it blank exposes your webhook endpoints to unauthorized access. Poster Forge utilizes safe path translations, but running the container behind a reverse proxy (like Nginx or Traefik) without basic auth is not recommended for external facing networks.
